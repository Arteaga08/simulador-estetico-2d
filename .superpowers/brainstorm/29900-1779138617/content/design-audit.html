Auditoría — Impeccable Skill
Lo que tenemos bien,
y lo que nos hace genéricos.
El skill "impeccable" tiene una lista explícita de patrones prohibidos para landing pages. Revisamos nuestras decisiones contra esa lista.
🚫
Inter está en la lista de rechazo de reflejos
Inter aparece explícitamente en el "reflex-reject list" de tipografías para superficies de marca (landing pages). Es la fuente más usada en SaaS de 2024-2026. Usarla en la landing hace que el diseño se vea como template de AI, no como marca propia. Para la app privada está bien. Para la landing, no.
BAN explícito AI slop
🚫
El gradiente azul→índigo es "AI slop" explícito
El bolder.md dice textualmente: "Avoid the purple-blue gradient AI slop." Nuestro #4FACFE → #667EEA es exactamente eso. En 2026 hay millones de landing pages con este gradiente. El color base #4FACFE puede quedarse. El gradiente como elemento identitario necesita evolucionar.
BAN explícito AI slop
🚫
Sección de stats = "hero-metric template" prohibido
Número grande + label pequeño + acento de gradiente. El skill lo llama explícitamente "SaaS cliché" y prohíbe usarlo. Nuestra sección de stats (60fps, 8+, 100%, 48h) es textbook de este patrón.
BAN absoluto
🚫
6 cards idénticas en Features = patrón prohibido
"Identical card grids. Same-sized cards with icon + heading + text, repeated endlessly." — prohibición absoluta. Nuestra sección de características tiene exactamente esto: 6 cards del mismo tamaño, icon + título + descripción.
BAN absoluto Template
⚠️
Labels "01 —" repetidos en cada sección = scaffolding de AI
"Repeated tiny uppercase tracked labels above every section heading. A single strong kicker can be voice; repeating it as section grammar is AI scaffolding." Está bien tener uno o dos, pero en cada sección se convierte en template.
Template
⚠️
Layout centrado en stack vertical = sin diseño
Todo centrado, columnas balanceadas, padding uniforme. El skill lo llama "splitting the difference into a generic centered stack." La landing necesita asimetría intencional, layouts 70/30, elementos que rompan el grid.
Template
💡
Sin imagen real en el hero
Para superficies de marca, el skill dice que "Zero images is a bug, not a design choice." Nuestro hero visual es un mock de UI, lo cual sí es válido para SaaS (product screenshot = imagery), pero necesita verse real y polished, no como un diagrama.
Por mejorar
💡
Sin motion ni animaciones de entrada
Una landing de marca puede tener "ambitious first-load motion, reveals, scroll-triggered transitions, typographic choreography." Actualmente todo es estático. En 2026, una landing sin ninguna animación se siente como un documento, no como un producto.
Por agregar
Qué cambiamos y cómo
Fix 01 — Tipografía
Landing: Geist + Geist Mono
Geist es la fuente de Vercel: no está en el reflex-reject list, tiene personalidad técnica-premium, tiene variable weight, y ya está disponible en Next.js como next/font/google. Aplicamos Geist para headlines de la landing. Para la app privada, Inter sigue siendo perfecto.
Landing headlines → Geist 800
App UI → Inter 400–700
Código/HUD → Geist Mono
Fix 02 — Color
Estrategia "Committed" con #4FACFE
#4FACFE como color dominante único (60% del hero), con neutros tintados hacia el azul. En lugar del gradiente azul→índigo como identidad, usamos #4FACFE sólido en grandes áreas y reservamos el gradiente solo para micro-elementos específicos.
Hero bg → #4FACFE sólido (drenched)
Texto sobre hero → #0A1628 (neutro tintado)
Gradiente → solo badges y sliders
Fix 03 — Features
Layout alternado, no grid de cards
En lugar de 6 cards idénticas, usamos 2-3 features grandes en layout alternado (texto izquierda + visual derecha, luego invertido), más una fila de 3 features menores. Cada feature tiene su propio tratamiento visual.
Feature principal → full-width alternado
Features secundarias → lista con icono
Sin cards idénticas
Fix 04 — Stats
Reemplazar métricas por prueba social narrativa
En lugar de "60fps / 8+ / 100%", comunicamos los mismos datos como afirmaciones de valor: una franja de texto prominente con las ventajas en formato conversacional, no como dashboard de números.
"Deformación fluida a 60fps — sin
pixelización, sin lag, sin Photoshop."

Datos como prosa, no como métricas.
Fix 05 — Layout
Asimetría intencional, hero 60/40
Hero con proporción 60/40: texto grande a la izquierda sin container centrado, visual del simulador a la derecha que rompe el grid. Algunas secciones sin container (full-bleed). Espaciado variado para crear ritmo.
Hero → 60/40, texto left-aligned
CTA section → full-bleed gradient
Espaciado variado: 48px / 80px / 120px
Fix 06 — Motion
Scroll reveals + entrada del hero
Animación de entrada del hero (título aparece letra a letra o por líneas), secciones con fade-in al hacer scroll, el simulador mock con animación de los sliders moviendose. Ease-out-quart, sin bounce.
Hero title → stagger por línea (300ms)
Secciones → fade + translateY(20px)
Simulador → sliders animated loop
Lo que NO cambia: #4FACFE como color principal, la paleta índigo para la app, Phosphor Icons, la estructura de 9 secciones de contenido, la navbar con gradiente, Inter para la app privada. Solo evolucionamos la expresión visual de la landing para que no se parezca a 10,000 otras landing pages de SaaS.
